# Messenger Web

Одностраничное приложение на React + TypeScript для работы с бэкендом мессенджера. Интерфейс обновлён: живая лента сообщений, список диалогов с быстрым созданием чата, загрузка вложений, профиль с аватаром.

## Быстрый старт

```bash
npm install
npm run dev        # dev-сервер на http://localhost:5173
```

Выберите конфиг для подключения к нужному backend (пример в `.env.example`):

- `.env.localhost` — http://127.0.0.1:8080 (по умолчанию с dev-прокси)
- `.env.dev100` — http://192.168.1.100:8080 (dev-прокси)
- `.env.dev101` — http://192.168.1.101:8080 (dev-прокси)

## Скрипты

- `npm run dev` — обычный dev-сервер
- `npm run build` — production сборка
- `npm run preview` — предпросмотр сборки
- `npm run lint` — ESLint

## Архитектура (коротко)

- `src/router` — роутер и защита по авторизации
- `src/features/auth` — контекст и хук авторизации
- `src/features/chats` — список чатов и окно сообщений с WebSocket
- `src/pages` — страницы входа, регистрации, чатов и профиля
- `src/shared/api` — HTTP клиент, типы и эндпоинты
- `src/styles` — тема и набор утилитарных стилей

## Важные моменты

- Все запросы идут с `credentials: 'include'`. В dev по умолчанию включён прокси Vite (флаги `VITE_USE_PROXY=true` + `VITE_PROXY_TARGET`), поэтому CORS не нужен.
- WebSocket URL формируется на базе `VITE_API_BASE_URL`, путь — `/ws/chat/:id`.
- Поддерживаются ответы, редактирование и удаление сообщений; вложения уходят сразу после подтверждения сервером сообщения.
